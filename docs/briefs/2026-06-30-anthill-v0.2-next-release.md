# anthill v0.2 — Next Release Bundle

**Created:** 2026-06-30\
**Status:** Active

---

## The Spark

Slice 1 shipped, anthill went public (github.com/ichabodcole/anthill), and it was
immediately validated on a _real foreign repo_ — a multi-app project (Elysia API + 2
Nuxt apps + Expo mobile + an SDK) bootstrapped by an external team ("media-buffet",
lead `arthur`). The methodology held and even over-delivered, but the run plus a
series of design conversations surfaced a coherent set of changes that belong together
as the next release. This brief captures that bundle and the decisions already locked,
so the anthill-internal dev agent can pick each up and drive it through proposal → plan
without re-litigating settled questions.

## Vision

v0.2 makes anthill (a) cohesive and portable in its repo footprint, (b) upgradeable
across breaking releases, (c) usable by humans at a terminal (not just agents), (d)
self-reflective about team structure, and (e) genuinely good at multi-surface repos —
the shape real projects actually have. It also closes the loop on the original goal:
dream-flute migrates onto anthill and becomes its first internal consumer (the virtuous
dogfood cycle).

## The Features (bundle — locked decisions captured)

### 1. Consolidate everything under `.anthill/`

Kill both the `.team/` config-dir name and the `docs/team/` living-docs location. One
self-named root holds everything.

- **Layout:** `.anthill/{config.json, README.md (SOP), paper-cuts.md, dev/{seams.md, README.md, <handle>.md}, scratch/ (gitignored)}`.
- **Why:** portability — a `docs/` convention isn't universal, but one self-named root
  is identical in every repo (anthill drops into ANY project); single add/remove
  footprint; gitignore shrinks to one line `.anthill/scratch/`.
- **Locked:** a committed dotfolder is well-precedented (`.changeset/`, `.github/`).
  Keep the config `paths` **override** as an escape hatch (a repo that prefers
  `docs/team/` — e.g. dream-flute by habit — can point there). bootstrap/init drop a
  CLAUDE.md/AGENTS.md **pointer** for human discoverability (project-docs does this).
- **Ships as migration v1→v2** (rename dir + relocate living docs; `git mv`, history
  preserved) — the inaugural case for feature 2.

### 2. Versioned upgrade mechanic

When a breaking release changes the consumer-repo files anthill created, consumers need
a guided upgrade.

- **Modeled on** `project-docs:update-project-docs` (version stamp in a consumer file +
  ordered, never-skip `migrations/vN-to-vM` + release-please-kept version).
- **Locked (brain/hands split):** stamp a `version` field in `.anthill/config.json`
  (foundational — defines v1). **`anthill migrate`** = the deterministic hands
  (pure/tested migration fns: `git mv`, rewrite refs, bump stamp, fix gitignore).
  **`anthill:upgrade`** skill = the brain (detect repo-vs-plugin version → find
  migrations → dry-run → apply via CLI → handle JUDGMENT steps the CLI can't, esp.
  reconciling USER-EDITED living docs without clobbering → verify).
- **Both, not either** (skill-only = non-deterministic; CLI-only = can't merge
  hand-edited docs).

### 3. Global `anthill` CLI + lifecycle baked into the skills

The missing terminal door — a human had no way to `anthill attach` themselves (hit live;
had to `tmux attach` manually). The lead also drove whole sessions from outside the panes
on full `bun <plugin-path>` invocations.

- **Locked: global, NOT a repo-local script.** flute's `scripts/flute`+package.json
  pattern assumes a Bun/JS host; anthill targets ANY-language repos. `config.ts` already
  walks up to the marker = exactly what a global cmd needs, so zero-repo-footprint is
  preserved.
- **Build:** add a `bin` field to package.json (`anthill`→cli.ts) so
  `bun add -g github:ichabodcole/anthill` yields a global `anthill`; plugin skills
  resolve + call that same command (reuse the highest-semver resolver). One CLI, same
  verbs, humans + agents. Add lead-facing wrappers for the vine/board (the lead's
  clunkiest mechanical bit).
- **Lifecycle in skills:** bootstrap preflight detects the CLI and if absent **OFFERS to
  install** (consent-gated — a global install touches the user's machine, NEVER silent);
  `anthill:upgrade` bumps the global bin alongside repo migrations; uninstall =
  documented agent capability (`bun remove -g anthill`), not its own skill.

### 4. Structural team-fit reflection in the rituals

Sharpen principle 3 ("anthill adapts to the work") into an explicit prompt that
interrogates seat SCOPING/SHAPE, not just headcount.

- **The move:** "did a seat's real work diverge from its stated scope? split / merge /
  redraw a scope? is the role still the right shape?" = knock down the corridor, re-form
  the walls.
- **Two touchpoints:** finalize = backward/evidence-based (the session's friction IS the
  pheromone signal → capture proposed `seats[]` edits + the why); convene = forward
  ("is this the right team for the work in front of us?" → act on accumulated signal).
  Loop closes: finalize records the misfit → convene reshapes.
- **Field-confirmed:** the finalize ritual already asks "did the composition fit" but
  reads as add/remove/split; the explicit RE-SCOPE prompt is the gap to add.

### 5. Multi-surface archetype + smarter discovery (the headline)

The #1 field friction: the only archetype (layered-app) seats horizontal layers and
assumes ONE app; on a multi-surface repo it would force one "surface" seat spanning Vue
AND React Native (unrelated expertise tracks).

- **Locked — bootstrap output = the SEAM DECISIONS made visible, not a roster.** After
  detecting `apps/*` + `packages/*` and sniffing each stack, emit 2-3 candidate seatings,
  each fold/split annotated with ONE clause of WHY (seam strength): A=vertical
  (recommended for multi-surface), B=layered (+ the spanning-warning), C=lean/merged. A
  visible decision collapses the human's job to "yep / swap / merge" (ratification, not
  reconstruction) → one-pass convergence.
- **Decomposition heuristic:** "where can two people work SEMI-INDEPENDENTLY against a
  STABLE CONTRACT?" — strong seams (package/app/SDK-contract boundaries) → split; weak
  seams (two apps, same stack/shell) → fold. Default vertical (by-surface) when
  `apps/*`/workspaces detected; offer layered as the alt. verify = a by-work-type seat
  among by-surface seats.
- **Themed naming as a first-class affordance:** bootstrap offers a naming theme mapped
  onto roles (Arthurian on media-buffet; craft/optics on dream-flute), handles stay
  free-form. Seats feel like characters with ownership — on-thesis for durable seats /
  ephemeral agents.
- Grounding-doc detection (see backlog) belongs to this discovery work.

### 6. dream-flute dogfood migration (the real parity test)

anthill was extracted from dream-flute; dream-flute becoming a normal anthill consumer
proves the extraction lost nothing.

- **Locked:** let anthill SUPERSEDE the bespoke `feature/team-spawn` work (don't finish
  it). Virtuous cycle: consume anthill → hit friction → fix anthill repo → release →
  re-consume.
- **Shape:** parity-audit flute's team CLI/skills vs anthill → author flute's
  `.anthill/config.json` (real roster maestro + loom/mosaic/fathom/prism) → reconcile
  existing living docs WITHOUT losing accumulated content (the delicate part; the `paths`
  override can keep them at `docs/team/` if preferred) → retire `scripts/flute` team-* +
  the bespoke skills → verify via a real convene. Rides on top of features 1-3.

## How They Interlock (one coherent release)

`.anthill/` consolidation (1) _is_ migration v1→v2, which is the first thing the upgrade
mechanic (2) runs, which the global CLI (3) exposes to humans. The reflection refinement
(4) and multi-surface discovery (5) share a vocabulary (by-surface / by-work-type / mix).
The dream-flute migration (6) consumes 1-3 as its first real exercise. Rough sequence:
1 → 2 → 3 (foundation), then 4 + 5 (skill/discovery, parallelizable), then 6.

## What It Is / What It Isn't

**It is:** the post-Slice-1 hardening + portability + usability release, grounded in real
field use.

**It is not:** a from-scratch redesign. The Slice-1 kit, the three principles, and the
brain/hands split all stand; this sharpens and extends them.
